## Regras de Organização

- A raiz deve conter apenas arquivos essenciais.
- Relatórios operacionais não devem voltar para a raiz.
- Arquivos com segredos, banco local, logs, perfis de navegador e artefatos gerados devem permanecer fora do Git.
- Novas decisões permanentes devem ser registradas em `docs/changelog.md` ou no documento temático correspondente.
